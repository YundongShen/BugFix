## 测试用例设计

```text
Test Case ID: TC_001
Description: 验证用户能够成功添加任务并查看反馈
Steps:
1. 用户进入主页，点击“添加任务”按钮。
2. 在弹出的输入框中填写任务内容（例如“完成报告”）并确认。
3. 观察任务列表的变化。
Expected Result: 
- 新任务出现在任务列表顶部。
- 任务卡片带有绿色动画效果。
- 数据库中新增一条状态为“未完成”的任务记录。

```

```text
Test Case ID: TC_002
Description: 验证用户能够标记任务为已完成并查看反馈
Steps:
1. 用户进入主页，确保任务列表中至少有一条未完成的任务。
2. 点击某任务左侧的复选框以标记其为已完成。
3. 观察任务卡片和列表的变化。
Expected Result: 
- 任务文字变为灰色并带删除线。
- 数据库中对应任务的状态更新为“已完成”。

```

```text
Test Case ID: TC_003
Description: 验证用户能够删除任务并查看反馈
Steps:
1. 用户进入主页，确保任务列表中至少有一条任务。
2. 点击某任务右侧的垃圾桶图标以删除该任务。
3. 观察任务列表和页面提示的变化。
Expected Result: 
- 被删除的任务从列表中淡出。
- 页面顶部显示提示“任务已删除”。
- 数据库中对应的任务记录被移除。

```

```text
Test Case ID: TC_004
Description: 验证“主页”菜单项加载逻辑
Steps:
1. 用户点击导航栏中的“主页”菜单项。
2. 观察页面内容和任务列表。
Expected Result: 
- 页面加载所有未完成的任务。
- “主页”菜单项高亮显示（例如加粗）。

```

```text
Test Case ID: TC_005
Description: 验证“已完成任务”菜单项加载逻辑
Steps:
1. 用户点击导航栏中的“已完成任务”菜单项。
2. 观察页面内容和任务列表。
Expected Result: 
- 页面加载所有已完成的任务。
- “已完成任务”菜单项高亮显示（例如加粗）。

```

```text
Test Case ID: TC_006
Description: 验证快速切换菜单项的功能
Steps:
1. 用户点击“主页”菜单项，观察页面内容。
2. 接着点击“已完成任务”菜单项，观察页面内容。
3. 再次点击“主页”菜单项，观察页面内容。
Expected Result: 
- 每次点击菜单项后，页面正确加载对应的任务列表。
- 当前菜单项始终高亮显示。
- 不需要重新登录或认证。

```

```text
Test Case ID: TC_007
Description: 验证任务列表的初始状态
Steps:
1. 用户首次访问主页。
2. 观察任务列表的初始状态。
Expected Result: 
- 任务列表默认加载所有未完成的任务。
- 如果没有任务，页面显示空白或提示“暂无任务”。

```

```text
Test Case ID: TC_008
Description: 验证任务卡片的UI样式
Steps:
1. 用户进入主页，观察任务卡片的外观。
2. 分别查看未完成任务和已完成任务的样式。
Expected Result: 
- 未完成任务：文字为黑色，无删除线。
- 已完成任务：文字为灰色，带删除线。
- 每张任务卡片左边有勾选框，右边有垃圾桶图标。
- 整体配色符合极简风格（白色背景，蓝色重点色）。

```

## 注意事项
- 测试环境需确保数据库已初始化，并包含若干测试数据（包括未完成和已完成任务）。
- 测试过程中需验证前端与后端的数据一致性。
- 所有测试用例应在不同分辨率和设备上进行兼容性测试。
